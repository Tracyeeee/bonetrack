## 一、产品概述

### 1.1 产品简介

**Bonetrack 骨迹**是一款专注于骨科康复管理的微信小程序，为骨科术后患者和保守治疗用户，提供从伤情记录、康复训练打卡到社区交流的一站式康复管理服务。

### 1.2 目标用户

| 用户类型 | 特征 | 需求 |
|---------|------|------|
| ACL/膝关节术后患者 | 已完成手术，处于康复期 | 记录康复进度、追踪关节活动度、管理训练任务 |
| 保守治疗用户 | 无需手术，物理康复 | 记录伤情、跟踪康复效果、获取康复指导 |
| 运动损伤康复者 | 踝关节、髋关节、肩关节等损伤 | 针对性康复训练、社区经验分享 |
| 脊柱损伤用户 | 颈椎、腰椎问题 | 日常康复管理、症状记录 |

### 1.3 核心价值主张

- **可视化康复进程**：通过日历、打卡、数据统计让康复进度一目了然
- **科学训练指导**：内置康复动作库，提供专业训练指导
- **社区互助支持**：与同病相怜的伤友交流，分享康复经验
- **隐私安全保护**：本地优先存储，云端同步备份

## 三、功能模块详细说明

### 3.1 首页（康复仪表盘）

#### 3.1.1 欢迎卡片

**位置**：页面顶部

**展示内容**：
- 用户昵称问候语
- 术后/受伤天数（大字号突出显示）
- 打卡天数统计
- 康复完成率

**交互行为**：
- 点击「术后/伤后天数」区域 → 跳转至日历页面
- 点击「打卡天数」区域 → 跳转至日历页面

#### 3.1.2 今日任务列表

**默认训练任务**（5项）：

| 任务名称 | 图标 | 分类 | 组数 | 次数/时长 |
|---------|------|------|------|-----------|
| 直腿抬高 | 🦵 | 力量训练 | 3组 | 10次 |
| 踝泵运动 | 🦶 | 关节活动 | 3组 | 20次 |
| 股四头肌收缩 | 💪 | 力量训练 | 3组 | 15次 |
| 膝关节屈伸 | 🔄 | 关节活动 | 3组 | 15次 |
| 平衡垫站立 | ⚖️ | 平衡训练 | 3组 | 1分钟 |

**自定义任务**：
- 支持添加自定义训练动作
- 可设置组数、次数/时长
- 支持设置有效期（开始日期-结束日期）

**任务交互**：
- 点击任务 → 切换完成/未完成状态
- 左滑或点击删除按钮 → 删除任务（需确认）
- 点击「+ 新增」→ 打开新增任务弹窗

**数据存储**：
- 任务状态按日期存储（`tasks_YYYY-MM-DD`）
- 自定义任务存储在 `customTasks`
- 已删除任务ID存储在 `deletedTaskIds`（全局过滤）

#### 3.1.3 快捷记录弹窗

**触发方式**：点击悬浮「+」按钮

**记录类型**：

1. **角度记录**
   - 输入框：数字输入，支持小数
   - 单位：度（°）
   - 可选：上传角度照片（拍照/相册）

2. **负重记录**
   - 输入框：数字输入，支持小数
   - 单位切换：kg / 斤
   - 快捷选项（近期负重值）

**保存逻辑**：
- 保存至 `checkin_YYYY-MM-DD` 存储
- 同时记录 `angle`、`anglePhotos`、`weight`、`weightUnit`

#### 3.1.4 悬浮打卡按钮

**样式**：右下角悬浮橙色圆形按钮
**图标**：+
**点击行为**：打开快捷记录弹窗

### 3.2 日历页面

#### 3.2.1 顶部统计卡片

**展示内容**：
- 术后/受伤总天数
- 累计打卡天数
- 康复完成率

#### 3.2.2 月度日历

**日期单元格状态**：
| 状态 | 样式 | 触发条件 |
|------|------|----------|
| 空单元格 | 灰色文字 | 无数据 |
| 今日 | 橙色边框高亮 | `isToday` |
| 有记录 | 圆点标记 | `hasRecord` |
| 有照片 | 照片图标 | `hasPhoto` |
| 有角度 | 角度值标签 | `angle > 0` |
| 完成训练 | "完成"标签 | `allTasksCompleted` |
| 未来日期 | 浅色文字 | `isFuture` |

**日期点击行为**：打开日期详情弹窗

#### 3.2.3 日期详情弹窗

**不同日期状态显示**：

1. **历史日期 - 有记录**
   - 训练任务完成状态列表
   - 角度照片预览（可点击放大）
   - 角度数据（如有）
   - 负重数据（如有）
   - 事项标签（如有）
   - 操作按钮：「补充打卡」「添加事项」

2. **历史日期 - 无记录**
   - 空状态提示
   - 「添加事项」「补充打卡」按钮

3. **今日 - 有记录**
   - 训练任务完成状态
   - 数据展示
   - 操作按钮

4. **今日 - 无记录**
   - 空状态提示
   - 「补充打卡」按钮

5. **未来日期**
   - 空状态提示
   - 「添加事项」按钮

**编辑模式**：
- 角度输入框
- 负重输入框（kg）
- 事项输入框
- 快捷标签：`复诊`、`换药`、`康复评估`、`复查`、`游泳`、`理疗`

**照片查看器**：
- 全屏图片预览
- 轮播切换
- 关闭按钮

#### 3.2.4 康复指南（动作库）

**分类标签**：
| 分类 | 图标 |
|------|------|
| 全部 | 📋 |
| 关节活动 | 🦵 |
| 力量训练 | 💪 |
| 平衡训练 | ⚖️ |
| 有氧训练 | 🏃 |

**动作卡片展示**：
- 动作图标
- 动作名称
- 难度标签
- 组数×次数
- 动作描述（截断）
- 动作要点提示

**动作详情弹窗**：
- 动作名称、难度、训练参数
- 动作说明
- 动作要点（带序号）
- 「返回」「去训练」按钮

### 3.3 广场页面

#### 3.3.1 帖子列表

**帖子卡片结构**：

```
┌─────────────────────────────────────┐
│ [头像] 用户名                        │
│ 部位 · 类型 · 原因 · 运动背景 · 天数  │
│                                     │
│ 帖子内容文字...                      │
│                                     │
│ [图片1] [图片2] [图片3]              │
│                                     │
│ #标签1 #标签2 #标签3                 │
│                                     │
│ ❤️ 123  💬 45        发布时间  删除  │
└─────────────────────────────────────┘
```

**自动生成标签**：
- `#损伤类型`（如 #ACL断裂）
- `#术后N天`（如 #术后45天）
- `#术后N天以上`（90天以上时）
- `#运动项目`（运动爱好者多选）
- `#术后早期` / `#康复中` / `#功能恢复期` / `#重返运动`

**交互行为**：
- 点击头像/用户名 → 查看用户主页（开发中）
- 点击图片 → 全屏预览
- 点击评论图标 → 展开/收起评论列表
- 点击「写评论...」→ 打开评论输入框
- 点击点赞图标 → 切换点赞状态
- 点击删除 → 删除帖子（仅自己发布的）

#### 3.3.2 筛选面板

**筛选维度**：

1. **关键词搜索**
   - 支持匹配：损伤类型、损伤部位、帖子内容、标签
   - 实时提示

2. **损伤部位**（三级联动）
   ```
   下肢 → 膝关节 → ACL断裂/PCL断裂/半月板撕裂...
   下肢 → 踝关节 → ATFL损伤/跟腱断裂...
   下肢 → 髋关节 → 盂唇撕裂...
   上肢 → 肩关节/肘关节/腕关节 → ...
   脊柱 → 颈椎/腰椎 → ...
   ```

3. **运动背景**
   - 专业运动员
   - 运动爱好者（可进一步筛选具体运动）
   - 偶尔运动
   - 久坐为主

**筛选操作**：
- 「重置」→ 清空所有筛选条件
- 「应用筛选」→ 执行筛选并关闭面板

**空结果处理**：
- 无符合条件帖子时显示空状态
- 提供「点击重置筛选」入口

#### 3.3.3 发布弹窗

**发布流程**：
1. 填写内容（必填，最多500字）
2. 添加图片（可选，最多9张）
3. 自动生成标签预览
4. 点击「发布」

**发布内容包含**：
```javascript
{
  authorId: openId,
  authorName: 昵称,
  authorAvatar: 头像URL,
  daysSinceSurgery: 术后天数,
  injuryPart: 损伤部位,
  injuryType: 损伤类型,
  injuryReason: 损伤原因,
  sportBackground: 运动背景,
  sportsItems: 运动项目列表,
  content: 帖子内容,
  images: 图片URL数组,
  tags: 自动生成的标签
}
```

**内容安全检测**：
- 文本内容安全检测（微信msgSecCheck）
- 图片内容安全检测（微信imgSecCheck）

#### 3.3.4 评论功能

**评论列表**：
- 显示评论用户名、内容、时间
- 「写评论...」输入入口

**评论输入**：
- 底部弹出输入框
- 支持发送按钮和键盘发送
- 发送后自动关闭

### 3.5 伤情简历

#### 3.5.1 步骤1：损伤部位

**部位大类选择**：
- 下肢（🦵）
- 上肢（💪）
- 脊柱（🧍）

**具体位置**（选择大类后显示）：
- 下肢：膝关节、踝关节、髋关节
- 上肢：肩关节、肘关节、腕关节
- 脊柱：颈椎、腰椎

**损伤类型**（选择具体位置后显示）：
- 膝关节：ACL断裂、PCL断裂、半月板撕裂、MCL损伤、LCL损伤、髌骨骨折、胫骨平台骨折...
- 踝关节：ATFL损伤、CFL损伤、下胫腓联合损伤、跟腱断裂...
- 以及其他部位对应类型

**自定义选项**：
- 每个层级都支持「+ 自定义」
- 自定义输入后需确认

#### 3.5.2 步骤2：治疗方式

**治疗方式选择**：
| 方式 | 图标 | 说明 |
|------|------|------|
| 手术治疗 | 🏥 | 已进行手术 |
| 保守治疗 | 💊 | 无需手术 |

**日期选择**：
- 手术日期 / 受伤日期选择器
- 快捷选项：今天、1周前、2周前、1个月前、3个月前

**天数显示**：
- 选择日期后显示截至今日的天数

#### 3.5.3 步骤3：基本信息

**必填项**：
- 昵称（文本输入）

**选填项**：
- 损伤原因：运动损伤、意外事故、工作相关、日常活动、疾病导致、自定义
- 运动背景：专业运动员、运动爱好者、偶尔运动、久坐为主

**运动爱好者扩展**：
- 喜欢的运动（多选）：滑雪、足球、篮球、攀岩、跑步、游泳、骑行、网球、其他

#### 3.5.4 步骤4：授权与完成

**简历摘要展示**：
- 损伤部位
- 损伤类型
- 治疗方式
- 手术/受伤日期
- 康复天数
- 昵称

**授权说明**：
- 打卡数据与微信账号绑定
- 开启广场发布权限
- 数据云端同步

**操作按钮**：
- 「微信授权并完成」

### 3.7 云函数接口

#### 3.7.1 getPosts（获取帖子列表）

**请求参数**：
```javascript
{
  page: 1,           // 页码，默认1
  pageSize: 10,      // 每页数量，默认10
  injuryPart: '',    // 损伤部位筛选
  injuryType: '',    // 损伤类型筛选
  sportBackground: '',// 运动背景筛选
  sportsItems: [],   // 运动项目筛选
  keyword: '',       // 关键词搜索
  orderBy: 'createdAt', // 排序字段
  orderType: 'desc'     // 排序方向
}
```

**返回数据**：
```javascript
{
  success: true,
  data: {
    posts: [{
      _id, id, authorId, authorName, authorAvatar,
      daysSinceSurgery, injuryPart, injuryType,
      injuryReason, sportBackground, sportsItems,
      content, images, tags,
      likes, comments, isLiked,
      createTime, showComments, commentList
    }],
    pagination: {
      page, pageSize, total, totalPages, hasMore
    }
  }
}
```

#### 3.7.2 createPost（发布帖子）

**请求参数**：
```javascript
{
  content: '',        // 帖子内容（必填）
  images: [],         // 图片URL数组
  authorName: '',     // 作者昵称
  authorAvatar: '',   // 作者头像
  daysSinceSurgery: 0,// 术后天数
  injuryPart: '',     // 损伤部位
  injuryType: '',     // 损伤类型
  injuryReason: '',   // 损伤原因
  sportBackground: '',// 运动背景
  sportsItems: []    // 运动项目
}
```

**返回数据**：
```javascript
{
  success: true,
  data: {
    _id,
    ...postData,
    createdAt
  }
}
```

#### 3.7.3 deletePost（删除帖子）

**请求参数**：
```javascript
{
  postId: ''  // 帖子ID
}
```

**返回数据**：
```javascript
{
  success: true,
  message: '删除成功'
}
```

**权限验证**：
- 仅帖子作者可删除

#### 3.7.4 comment（评论功能）

**请求参数**：
```javascript
{
  action: 'add' | 'list' | 'delete',
  postId: '',      // 帖子ID
  commentId: '',   // 评论ID（删除时需要）
  content: '',     // 评论内容（添加时需要）
  authorName: '',   // 评论者昵称
  authorAvatar: ''  // 评论者头像
}
```

**返回数据**：
```javascript
{
  success: true,
  data: {
    _id,
    postId, authorId, authorName, authorAvatar,
    content,
    createTime,
    // 添加时返回新评论
    // 列表时返回评论数组和分页信息
  }
}
```

#### 3.7.5 like（点赞功能）

**请求参数**：
